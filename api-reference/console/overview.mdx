The Console API is the admin surface that backs the HyperAuth dashboard. Every route lives under `/v1/console/*` on the orchestrator, is scoped to an organization and app, and is gated by a UCAN-based capability. Use it to manage operator-owned resources programmatically without going through the dashboard UI.

## When to use it

- You are building tooling that automates app, user, or credential management.
- You need to script API key or webhook rotation as part of a deploy pipeline.
- You want to export audit events into your own SIEM or analytics store.

If you are integrating end-user sign-in into a product, use the [SDK](/sdk-reference/sdk-client) and the public [Registration](/api-reference/registration/sessions) and [Email-code](/api-reference/email-code/start) endpoints instead.

## Path structure

All resources are nested under an organization and app:

```
/v1/console/orgs/:orgId/apps/:appId/<resource>
```

The two exceptions are top-level org and member routes, which omit the `apps/:appId` segment.

## Authentication

Every request must carry a UCAN bearer token in the `Authorization` header:

```http
Authorization: Bearer <ucan-jwt>
```

Each route requires a specific capability on the principal — for example `console.api_key.write` to create an API key, or `console.audit.read` to list audit events. A token without the required capability gets a `403 forbidden` response.

| Capability prefix | Covers |
|---|---|
| `console.org.*` | Organizations |
| `console.member.*` | Org members |
| `console.app.*` | Apps |
| `console.end_user.*` | End users |
| `console.credential.*` | WebAuthn / OAuth credentials |
| `console.session.*` | End-user sessions |
| `console.api_key.*` | App API keys |
| `console.webhook.*` | Webhook endpoints |
| `console.policy.*` | Auth policies |
| `console.audit.*` | Audit events (read-only) |

Read routes use the `*_READ` capability, write routes use `*_WRITE`, deletes use `*_DELETE`, and session revoke uses `console.session.revoke`.

## Conventions

- **List responses** return `{ "rows": [...], "total": <int> }`. Use `?page=` (1-indexed) and `?pageSize=` (max `100`, default `25`) to paginate.
- **Dates** are ISO 8601 strings.
- **BigInt** fields (for example `sbtTokenId`, `tbBudgetAccountId`) are serialized as decimal strings to preserve precision.
- **Show-once secrets** — API key secrets and webhook signing secrets are returned only on create and rotate. They are never included in subsequent reads. Store them immediately.
- **Validation errors** return `400` with `{ "error": "validation_error", "details": <zod-format> }`.
- **Not found** returns `404` with `{ "error": "NOT_FOUND", "message": "<resource> <id> not found" }`.
- **Missing capability** returns `403 { "error": "forbidden" }`.

## Example: list end users

```bash
curl https://api.hyperauth.dev/v1/console/orgs/org_01H.../apps/app_01H.../end-users?page=1&pageSize=25 \
  -H "Authorization: Bearer $UCAN_TOKEN"
```

```json
{
  "rows": [
    {
      "id": "eu_01HZ...",

    }

}
```
